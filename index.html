Benchmark Domains
Controlled Horizon Extension

Tasks are constructed with systematically increasing H* via two extension methods.

🌐
Web Navigation
Based on WebArena. Tasks span e-commerce, CMS, GitLab, Reddit, and mapping. Breadth extension composes independent subtasks into composite workflows. Collapses earliest — at very small s.
Breadth Extension s = 1..4 WebArena base
💻
Operating System
Based on AgentBench. Shell-level control with file permission chains. Depth extension inserts non-skippable intermediate states — e.g., enforce read-only before making writable.
Depth Extension s = 1..8 AgentBench base
🗄️
Database
Custom environment built from scratch. 3-module agent: Selector → Decomposer → Refiner. Evaluated by Filter-k accuracy. Tasks range from simple queries to multi-table joins with nested sub-goals.
Depth Extension s = 1..8 Custom env
🤖
Embodied Manipulation
IsaacSim 5.0 with bimanual Franka Panda arms. Three objects: red, blue, green cubes. Four primitives: home, move_to_pose, grasp, release. Steepest degradation of all domains.
Breadth Extension s = 1..2 IsaacSim 5.0
Failure Attribution
7-Category Failure Taxonomy

Categories represent orthogonal dimensions of agent behavior — not mutually exclusive classes. A single failed trajectory may exhibit multiple failure types simultaneously. Validated by two domain experts via blind annotation and consensus discussion.

Community
Contribute to HORIZON

Evaluate your model and submit results to the leaderboard. All model families and research groups welcome.

Step 01 — Evaluate
Run Your Agent on HORIZON
Clone the evaluation harness and run across all four domains. Baseline tasks are pre-verified solvable, so performance degradation reflects genuine horizon difficulty — not task ambiguity.
git clone https://github.com/horizon-bench/horizon cd horizon && pip install -r requirements.txt # Evaluate across all domains python evaluate.py \ --agent your_agent \ --domains web os db embodied \ --output results.json
Step 02 — Submit
Open a Pull Request
Submit your results.json with per-task success rates and a model card. We validate and add you to the leaderboard.
  • Per-task success rates across all extension levels s
  • Model card: parameter count, context length, training details
  • Optionally annotate ≥20 failed trajectories with the 7-category taxonomy
  • Human or LLM-as-a-Judge annotations both accepted
Open a Pull Request →
Step 03 — Extend the Benchmark
Add New Tasks or Domains
HORIZON is designed to be extensible. Propose new baseline tasks or new domains following our horizon-extension protocol.
  • New tasks require an oracle/expert solution to establish H*
  • Depth extension: each added step must be non-skippable by an optimal policy
  • Breadth extension: H*(s) must be additive across composed subtasks
  • All 7 taxonomy categories must be representable in the new domain
Step 04 — Annotate Failures
Contribute Failure Analysis
Failure attribution is as important as success rate. Annotate failed trajectories to build a richer diagnostic picture for the community.
  • Use the 7-category taxonomy — categories are non-exclusive
  • LLM-as-a-Judge pipeline provided for scalable annotation
  • Grounded in trajectory context for reproducible analysis
  • Enables targeted interventions beyond model scaling

Abstract
Position: The Long-Horizon Task Mirage?
Diagnosing Where and Why Agentic Systems Break

We argue that identifying the breaking point of long-horizon agents is inherently challenging, but can be meaningfully studied through a structured diagnostic framework. We propose HORIZON, which generates tasks with increasing intrinsic horizons and quantifies how agent success degrades — using a seven-category failure taxonomy applied across web navigation, operating systems, databases, and embodied manipulation.
